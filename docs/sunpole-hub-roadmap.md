# SUNPOLE HUB — история работы, roadmap и список задач

## Текущий этап

Проект перешёл из стадии идеи в стадию рабочих прототипов.

Сейчас есть:

- старый `shop.html` как визуальная отправная точка;
- `hub.html` как первый технический прототип;
- `hub-v2.html` как первая визуальная версия;
- `hub-v3.html` как версия с улучшенным Project Passport и тегами;
- `hub-v4.html` как версия с рабочим сравнением проектов;
- `project-view.html` как первая страница отдельного проекта;
- `hub-versions.html` как переключатель контрольных HTML-версий;
- `data/projects-hub.js` как новая база проектов;
- `docs/hub-version.html` как текущая страница версии;
- `docs/sunpole-hub-vision.md` как концепция и видение.

## Главная цель

Создать личный визуальный магазин/хаб проектов `SUNPOLE HUB`, который работает как:

- каталог проектов;
- витрина;
- технический паспорт;
- история версий;
- инструмент сравнения;
- админка;
- локальный автоматизатор базы и скриншотов;
- система аккуратного переноса старых проектов без потери ссылок.

### Project Passport

- Реализован первый Project Passport через кнопку `⋯`.
- В `hub-v3.html` и `hub-v4.html` Passport открывается рядом с карточкой.
- Повторное нажатие по `⋯` закрывает Passport.
- В Passport отображаются:
  - название проекта;
  - ID;
  - категория;
  - версия HUB;
  - качество;
  - HP-бары;
  - языки;
  - теги;
  - связанные версии;
  - кнопка открытия проекта;
  - ссылка на версию HUB.

### Сравнение проектов

- В `hub-v4.html` реализован первый рабочий режим сравнения.
- Кнопка `⇄` выбирает проект для сравнения.
- Можно выбрать до 4 проектов.
- Выбранные карточки подсвечиваются.
- Появляется нижняя панель сравнения.
- Сравниваются:
  - статус;
  - категория;
  - качество;
  - готовность;
  - визуал;
  - техника;
  - документация;
  - теги;
  - языки;
  - связанные версии.

### Project View

- Создан `project-view.html`.

- На странице проекта есть:

  - название;
  - статус;
  - категория;
  - описание;
  - теги;
  - качество;
  - HP-бары;
  - языки;
  - кнопка открытия проекта;
  - кнопка GitHub;
  - похожие и связанные проекты.

### Страница переезда проекта

Нужен шаблон:

```text
moved.html
```

или

```text
project-moved.html
```

Он должен показывать:

- сообщение: проект переехал;
- старый адрес;
- новый адрес;
- кнопку открытия нового адреса;
- кнопку перехода в HUB;
- предложение обновить закладку;
- возможно автоматическое перенаправление через несколько секунд.

Это важно, потому что некоторыми старыми проектами могли пользоваться друзья или знакомые.

### Безопасная миграция проектов

Пока нельзя массово переносить папки.

Перед переносом нужно:

1. Создать backup-ветку:

```text
backup-before-structure-move
```

2. Создать карту старых и новых адресов.
3. Протестировать переезд на одном проекте.
4. Добавить moved-страницу.
5. Только потом переносить проекты.

## Важные принципы

- Ничего не удалять без отдельного решения.
- Старый `shop.html` не удалять.
- Старые версии проектов не удалять.
- Архив — это статус, а не удаление.
- Новые идеи фиксировать в документации.
- Крупные изменения делать контрольными версиями.
- Перед переносом файлов создать backup-ветку.
- Не менять старые ссылки без redirect/moved-страниц.
