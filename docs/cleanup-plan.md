# STILIST Cleanup Plan

## Цель
Превратить репозиторий в удобный каталог проектов без потери существующих файлов.

## Правила безопасности
- Ничего не удалять на первом этапе.
- Ничего не переименовывать на первом этапе.
- Не переносить проекты между папками до завершения аудита.
- Использовать shop.html как основную витрину проектов.

## Этап 1. Инвентаризация
- Найти все HTML-проекты.
- Определить назначение каждого проекта.
- Составить список категорий.

## Этап 2. Каталог
Предварительные категории:
- Полиграфия
- Игры
- Церковь
- Дизайн
- Инструменты
- Файлы и JSON
- Эксперименты
- Архив

## Этап 3. Витрина
- Использовать shop.html как единый каталог.
- Добавить статусы проектов.
- Добавить поиск и фильтры по категориям.

## Этап 4. Архив
- Отметить старые проекты как архивные.
- Не удалять архивные проекты.

## Этап 5. Реорганизация
Только после полного аудита рассматривать перенос файлов и изменение структуры каталогов.

## Текущий статус
Создан первоначальный план. Следующий шаг — аудит проектов и составление карты репозитория.
