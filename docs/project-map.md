# STILIST Project Map

## Назначение файла
Это первичная карта проектов репозитория `sunpole/stilist`. Файл создан для наведения порядка без удаления, переименования и переноса существующих файлов.

## Главные файлы

| Файл | Роль | Комментарий |
|---|---|---|
| `index.html` | старая навигация | Генерирует ссылки на `index2.html` — `index130.html`. Полезно как временный технический список. |
| `shop.html` | главная витрина проектов | Лучший кандидат на основной HUB. Уже содержит карточки, поиск, теги и ссылки на проекты. |
| `c-shop.html` | конструктор карточек | Простая версия конструктора карточек для витрины. |
| `c2-shop.html` | продвинутый конструктор карточек | Более полезная версия: карточки с тегами и скриншотами. Оставить как служебный инструмент. |
| `README.md` | описание репозитория | Сейчас очень короткое. Позже стоит расширить. |

## Что важно не делать сейчас

- Не удалять старые `index*.html`.
- Не переносить папки.
- Не переименовывать файлы.
- Не заменять `index.html` до полной проверки.

## Рекомендованный следующий шаг

1. Продолжить аудит всех проектов из `shop.html`.
2. Отметить дубликаты и версии.
3. Определить основной вариант для похожих проектов:
   - `3501` / `3502`
   - `80` / `81`
   - `index120.html` / `400/`
   - `index5.html` / `5/`
4. Потом вынести массив проектов из `shop.html` в отдельный файл `data/projects.js` или `data/projects.json`.
5. Только после этого улучшать внешний вид `shop.html`.

Главный рабочий путь — развивать `shop.html` как `SUNPOLE HUB`. Текущую структуру пока лучше не ломать. Сначала надо сделать понятную витрину и карту проектов.
